# Roadmap to true DAx

The current model is a post-action short-horizon attacking threat component. A true DAx system requires:

1. Pre-action threat estimation.
2. Post-action threat estimation.
3. Counterfactual threat without intervention.
4. Option availability before the action.
5. Option removal after the action.
6. Multi-defender attribution.
7. Role and team adjustment.
8. Opportunity adjustment.
9. Human football validation.
10. Uncertainty estimates.

Future observed baseline:

`DAx observed baseline = predicted threat immediately before the action - predicted threat immediately after the action`

Future counterfactual version:

`DAx counterfactual = expected threat without the defensive intervention - expected threat after the observed intervention`

No fake counterfactual values are implemented.
