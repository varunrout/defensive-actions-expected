# Modelling limitations

This draft is still provisional and must not be interpreted as true DAx or causal player value.

Known limitations in this environment:

- `data/features/player_defensive_actions.parquet` is absent, so real-data execution could not be completed here.
- Package installation is blocked by the package index/network (`403 Forbidden` while fetching build dependencies), so actual MLflow-enabled integration could not be run in this container.
- MLflow behaviour is covered by unit tests with a fake installed module and by clear unavailable/unreachable failure tests; a real local tracking-store test should be run once the `models` extra is installable.
- Calibration support exists for fold-internal Platt/isotonic fitting, but the current production comparison defaults to uncalibrated variant rows unless expanded to run multiple calibration modes as separate candidate rows.
- Robustness slices should be reviewed on real data before player-signal outputs are shared.

No output generated by this workflow should be called true DAx.
